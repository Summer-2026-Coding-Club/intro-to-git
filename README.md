# Git Mini-Course

This folder is a small Git course for students with different experience levels.

[Summer-2026-Coding-Club](https://github.com/Summer-2026-Coding-Club)

## Shared Demo

Across all three levels, students work on the same project:

```text
NAME-time-capsule
```

Example:

```text
marco-time-capsule
secret-time-capsule
the-end-of-time-capsule
```

The repo contains a `README.md` with things like:

- a project codename
- a tiny status update
- a ridiculous app idea
- a challenge for visitors

## Course Map

| Level | Who it is for | What students learn | Demo milestone |
| --- | --- | --- | --- |
| [Level 1](./git-level-1.md) | New to Git and GitHub | Basic terms: Git, GitHub, repo, commit, push, pull, fetch | Create a repo in the org and edit the README on GitHub |
| [Level 2](./git-level-2.md) | Has seen Git but is not confident | Install/setup Git, clone, branch, status, add, commit, push, pull | Clone the repo, make a branch, edit locally, push the branch |
| [Level 3](./git-level-3.md) | Knows the basics | Pull requests, diffs, merge, conflicts, cleanup habits | Open a PR, inspect the diff, merge it, sync local `main` |

## My Take

Git can feel confusing because people explain too many commands at once. For beginners, I think Git makes more sense when you use it for something small first.

The first win is simple:

1. Make a repo.
2. Change a file.
3. Save the change as a commit.
4. Push it to GitHub.
5. Let someone else see it.

Everything else builds on that.
